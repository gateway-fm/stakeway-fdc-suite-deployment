# Stakeway FDC Suite Deployment Notes

## Architecture

This is a fork of [flare-foundation/fdc-suite-deployment](https://github.com/flare-foundation/fdc-suite-deployment).
Stakeway-specific changes are applied on the `stakeway-fork-2` branch. Port and container name
changes are made directly in compose files. Additive changes (extra_hosts, networks, dns) use
`docker-compose.override.yml` files.

## Branch Strategy

- `main` — synced with upstream, no Stakeway changes
- `stakeway-fork-2` — active branch with Stakeway customizations for v1.2.4 parallel deployment

## Customization Pattern

Each service directory may have:
- `docker-compose.yaml` — modified from upstream (ports, container_name only)
- `docker-compose.override.yml` — additive Stakeway config (extra_hosts, networks, dns)
- `*.local.conf` — server-only config files with credentials (gitignored)

## Cutover Procedure

When -2 indexing is complete:
1. Stop -1 verifiers and nodes
2. Update -2 compose files: change ports back to production values, remove `-2` suffix from container names and project names
3. Restart -2 services
4. Verify all services healthy on production ports
5. Remove -1 directory and old Docker volumes

## Server-Only Files (NOT in git)

These must be created manually on each server:
- `.env` — node URLs, credentials, DB passwords
- `nodes-mainnet/btc/bitcoin.local.conf` — RPC auth credentials
- `nodes-mainnet/doge/dogecoin.local.conf` — RPC auth + rpcbind
- `nodes-mainnet/xrp/rippled.local.conf` — admin IP bindings for Docker network
- `verifiers/xrp/config.toml` — generated by generate-config.sh (already gitignored)
- Various `*.env` files — generated by generate-config.sh (already gitignored)

## Upstream Sync Workflow

GitHub Action runs weekly to detect upstream changes and create a PR.
When a sync PR arrives:
1. Review: check if upstream changed base docker-compose.yaml structure
2. Resolve any conflicts on port/container_name lines (trivial)
3. Merge the PR
4. On server: `cd /data/stakeway-fdc-suite-deployment-2 && git pull`
5. Validate: run `docker compose config` in each service directory
6. Restart affected services: `docker compose up -d`

## Key Customizations

1. **Port binding to specific IPs**: BTC node binds to 10.75.123.150 (not 0.0.0.0)
2. **XRP admin port**: Bound to 127.0.0.1 only
3. **Docker networks**: DOGE node/verifier uses `gateway-shared` external network
4. **extra_hosts**: All verifier services get `host.docker.internal:host-gateway`
5. **DNS**: Added to web2-verifier, BTC indexer, XRP indexer/node
6. **Config file mounts**: Local .conf files with credentials mounted over upstream defaults

## Docker Compose v2.21 Limitation

The server runs Docker Compose v2.21.0. Override files APPEND to port lists rather than replacing
them. The `!reset` and `!override` syntax is not supported. This is why ports and container_name
are modified directly in the base compose files rather than via overrides.
